# Contributing to ClarityDesk

## Branch Naming
\`\`\`
feature/short-description   ← new functionality
fix/short-description       ← bug fixes
chore/short-description     ← deps, configs, refactors
docs/short-description      ← documentation only
test/short-description      ← adding/updating tests
\`\`\`

## Commit Convention (Conventional Commits)
\`\`\`
feat(scope): add new feature
fix(scope): fix a bug
chore(scope): maintenance task
docs(scope): documentation update
test(scope): add or update tests
refactor(scope): code restructure
perf(scope): performance improvement
\`\`\`

## PR Process
1. Always branch off develop
2. Make your changes with clean commits
3. Open PR → develop
4. CI pipeline must pass
5. Merge with squash commit
